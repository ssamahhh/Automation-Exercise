# Test Plan – Automation Exercise Website

## 1. Project Information

**Project Name:** Automation Exercise Website Testing

Website: https://automationexercise.com

Prepared By: Samah Sameh

Testing Types: Manual Testing & Automation Testing

Version: 1.0

---

## 2. Objective

To verify that the Automation Exercise website functions correctly, meets business requirements, and provides a stable user experience through both manual and automated testing.

---

## 3. Scope

### In Scope

* User Registration
* User Login & Logout
* Product Search
* Product Details
* Shopping Cart
* Contact Us Form
* Subscription Feature
* Navigation Between Pages
* Checkout & Payment

### Out of Scope

* Performance Testing
* Security Testing
* Mobile Application Testing
* Third-party Integrations

---

## 4. Test Strategy

### Manual Testing

* Functional Testing
* UI Testing
* Regression Testing
* Exploratory Testing

### Automation Testing

Automated using:

* Java
* Selenium WebDriver
* TestNG
* Maven
* Allure Reports
* Page Object Model (POM)

Automated test scenarios include:

* Registration
* Login
* Logout
* Product Search
* Product Details
* Add to Cart
* Contact Us
* Subscription Validation
* Checkout
* Payment

---

| Item             | Details            |

| IDE              | IntelliJ IDEA      |

---

## 6. Entry Criteria

* Test environment is available.
* Test cases is prepared
* Test data is prepared.
* Website is accessible.

---

## 7. Exit Criteria

* All planned test cases executed.
* Critical and High defects resolved.
* Automation suite executed successfully.
* Test summary report generated.

---

## 8. Deliverables

* Test Plan
* Test Cases
* Bug Reports
* Automation Framework
* Allure Test Report
* Test Execution Summary

---

## 9. Risks

* Website downtime.
* Changes in UI locators.
* Unstable test environment.
* Browser compatibility issues.

---

## 10. Approval

Testing activities are considered complete when all exit criteria are met and reports are reviewed.
